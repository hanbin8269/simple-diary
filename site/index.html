Most diary apps
want your data.

They sync your most private thoughts to someone else's cloud, put an account between you and your own words, and call it a feature. A diary deserves better defaults. Simple Diary has no server to send your words to, and no login to forget — there is simply nothing on our side, because there is no "our side."

No servers. No accounts. Nothing to leak.
🗂️

Everything stays on your machine

Each entry is a dated, plain Markdown file in a folder you can see, open, and copy — synced between your devices by your own iCloud Drive, not by us. Export everything to Markdown, text, or JSON anytime. If this app disappears tomorrow, your diary doesn't.

Yesterday stays behind Touch ID

Past entries are hidden by default and only unlock with your fingerprint — Face ID on iPhone. The lock snaps back every time the app launches, so a shared screen or a glance over your shoulder never shows more than today. Write like no one's reading. No one is.

Install

Two ways, pick whichever suits you.

DownloadEasy

  1. Open SimpleDiary.dmg and drag Simple Diary into the Applications folder.
  2. If Chrome flags the download, click ⌃ → Keep — it's just an uncommon file.
  3. On first launch, if macOS warns you: right-click → Open, or allow it under System Settings → Privacy & Security → "Open Anyway".

It's a personal project shipped without Apple notarization — that's all the warnings are about.

Build from sourceDev

No Xcode required — Command Line Tools are enough.

# from the project folder
./build.sh
open "build/Simple Diary.app"

# install to Applications
cp -R "build/Simple Diary.app" /Applications/
📱 On iPhone too — a companion iOS app reads the same iCloud folder, with the same Face ID lock. Not on the App Store yet, so it's a build-it-yourself affair with Xcode (see the ios/ folder in the repo).
